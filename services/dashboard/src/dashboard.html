Write a temporal analytics query in the DSL, deploy it, and the system materializes a push-based agent graph that processes events in real time. Each sub-computation runs as a durable Golem agent. Use the Deploy Metric tab to register your own metrics.

Example: Connection Induced Rebuffering Ratio (CIRR)

duration_where( has_existed(playerStateChange == "play") && !has_existed_within(playerStateChange == "seek", 5) && latest_event_to_state(playerStateChange) == "buffer" ) | aggregate(group_by(cdn), count, sum, avg)
1

Run the demo

Start Golem, Kafka, generate events, and open this dashboard.

cargo make demo
2

Deploy a metric

Switch to the Deploy Metric tab. Enter a name and DSL expression, then click Deploy. The system compiles the expression, spawns a feeder that consumes from Kafka, initializes timeline agents per session, and routes events to the correct leaves automatically.

3

Stream sub-computations

Switch to the Computation Progress tab. Select a registered metric, enter a session ID (e.g., sess-0-cirr) and query time, then click ▶ Stream to watch every agent's timeline update live as events flow in.

4

Watch aggregation results

Switch to the Live Dashboard tab. Enter an aggregator agent name (e.g., aggregator-cdn-akamai) and enable auto-poll to watch cross-session metrics update in real time.
